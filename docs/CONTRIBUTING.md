# Contributing Guide

## 브랜치 전략(GitHub Flow)

- `main`: stable branch.
- `feature/<name>-<topic>`: work branch.

## 브랜치 네이밍 규칙

- 예: `feature/<name>-<topic>`

## 커밋 메시지 컨벤션

- 예: `feat: ...`
- 예: `fix: ...`
- 예: `docs: ...`
- 예: `test: ...`
- 금지 예: `update`, `temp`, `final`

## PR 규칙

- `Closes #<issue_number>`
- What changed
- Why it changed
- How it was tested

## 코드 리뷰 규칙

- `LGTM`만 남기는 리뷰는 충분하지 않습니다.

## 충돌 대응 흐름

- 충돌 발생 → 공유 → 해결 → 기록(`docs/conflict-resolution.md`)
